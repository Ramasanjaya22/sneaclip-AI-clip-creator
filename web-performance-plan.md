Laporan Peningkatan Keamanan (Security Hardening)
1. **Secret Handling & Session Safety**: Menambahkan properti `app.secret_key` dengan nilai acak kriptografis `os.urandom(32)` sebagai fallback apabila `FLASK_SECRET_KEY` environment variable tidak diset, memastikan cookies sesi dapat dienkripsi. `SESSION_COOKIE_SECURE`, `SESSION_COOKIE_HTTPONLY`, dan `SESSION_COOKIE_SAMESITE` ditambahkan dengan dependensi flag `is_prod`.
2. **Security Headers**: Memodifikasi helper middleware `set_cache_headers` dengan menyertakan `Content-Security-Policy` yang dikonfigurasi longgar tapi aman (hanya untuk CDN dan aset lokal) serta `Referrer-Policy: strict-origin-when-cross-origin` untuk mereduksi risiko Cross-Site Scripting (XSS).
3. **Input Validation & Extention Checking**: Route-route unggahan di Flask (video, music, dan watermark) telah dilengkapi dengan filter list ekstensi yang dilegalkan (`allowed_file_extension`), mencegah pengunggahan dan eksekusi potensial file jahat (seperti `.py`, `.exe`, dll).
4. **Path & Output Safety**: Parameter pengenal file/potongan unggahan rawan (chunk) seperti `upload_id` diamankan melalui sanitasi dari `secure_filename`. Penanganan eksepsi internal aplikasi dibuat terisolir dari mata pengguna awam lewat function helper `safe_error`, yang mengembalikan pesan generik 'Internal Server Error' bila di Production dan merekam raw traces/format log hanya di terminal dan `app.log`.
5. **Dampak Performa**: Sangat ringan hingga *zero-latency*. Perubahan dilakukan tanpa dependensi asing besar; menggunakan modul bawaan native yang sudah ter-import, mempertahankan alur kinerja dan pengalaman UI Sneaclip persis sediakala, meningkatkan stabilitas dari bad request yang tidak tervalidasi.
